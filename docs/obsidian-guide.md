# 내 Obsidian으로 Hachimon 쓰기

Hachimon은 **서버도 계정도 없는** 개인용 복습 앱입니다. 당신의 Obsidian 노트를 브라우저에서 직접 읽어 카드로 만들고, 복습 일정은 당신 브라우저(IndexedDB)에만 저장됩니다. **노트는 어디로도 업로드되지 않습니다.**

```
내 Obsidian Vault(.md)  →  앱에서 직접 가져오기  →  내 브라우저에만 저장
```

각자 자기 기기에서 앱을 설치하고, 자기 Vault를 연결해 쓰는 구조입니다.

---

## 1. 앱 설치 (PWA)

브라우저로 `https://hachimon.app` 에 접속한 뒤 홈 화면에 추가하면 앱처럼 쓸 수 있습니다.

- **iPhone (Safari)**: 공유 버튼 → `홈 화면에 추가`
- **Android (Chrome)**: 메뉴(⋮) → `앱 설치` / `홈 화면에 추가`
- **데스크톱 (Chrome/Edge)**: 주소창 오른쪽 설치 아이콘 클릭

설치하면 오프라인에서도 복습이 동작합니다.

> 설치 없이 브라우저 탭에서 바로 써도 됩니다. 데이터는 동일하게 그 브라우저에 저장됩니다.

---

## 2. 노트 작성 포맷

Hachimon은 각 노트의 **`## Self-Test Anchors`** 섹션 **아래만** 읽습니다. 그 윗부분(본문)은 무시되니, 평소처럼 노트를 쓰고 맨 아래에 복습 카드를 모아두면 됩니다.

````markdown
# Spring Transaction   ← 본문은 자유롭게. 파싱에서 무시됨

(여기에 평소 정리 내용…)

## Self-Test Anchors
#flashcard/spring/core

### Foundation
트랜잭션 전파란?::경계가 만났을 때 동작을 정하는 규칙.

### Mechanism
REQUIRES_NEW와 NESTED 차이는?::REQUIRES_NEW는 독립 트랜잭션이다.

### Diagnosis
롤백이 전파되지 않는 원인은?::별도 트랜잭션이라 전파되지 않는다.
````

### 규칙

| 요소 | 의미 |
|------|------|
| `## Self-Test Anchors` | 이 줄 **아래만** 카드로 스캔 |
| `#flashcard/스프링/코어` | 덱 경로. 슬래시(`/`)로 계층 구분 |
| `### Foundation` / `### Mechanism` / `### Diagnosis` | 난이도 티어 (대소문자 무관) |
| `질문?::답변` | `::` 앞은 질문, 뒤는 답변. 답변은 다음 카드/티어/헤딩 전까지 **여러 줄** 가능 (아래 "답변 마크다운" 참고) |

### 3-tier 난이도

- **Foundation** (파랑) — 개념·정의·용어
- **Mechanism** (앰버) — 동작 원리·내부 구현·비교
- **Diagnosis** (빨강) — 실전 진단·트러블슈팅·설계 판단

### 답변 마크다운 (여러 줄 가능)

답변은 **여러 줄**에 걸쳐 쓸 수 있습니다. `질문?::` 뒤(같은 줄이든 다음 줄부터든) 작성한 내용이 **다음 카드(`질문?::`)·다음 `### 티어`·다음 `#flashcard`·다음 헤딩·노트 끝** 전까지 한 답변으로 묶입니다. 코드블록·여러 문단·리스트가 그대로 보존되어 렌더링됩니다.

````markdown
### Mechanism
생성자 주입을 권장하는 이유는?::
- 불변(final) 보장, 테스트 용이
- 순환 의존을 컴파일 타임에 발견

```java
@Service
class OrderService {
    private final PaymentService pay;
    OrderService(PaymentService pay) { this.pay = pay; }
}
```

다음 질문?::이건 위 답변에 안 섞이고 새 카드가 됩니다.
````

- 코드블록은 ` ``` `(또는 `~~~`)로 감쌉니다. **펜스 안의 `::`·빈 줄은 카드 경계로 취급되지 않습니다.**
- 인라인 코드(백틱) 안의 `::`(예: `` `std::vector` `` · `` `key::value` ``)도 새 카드로 오인되지 않습니다.
- **단, 코드펜스/인라인코드 *밖*의 평문 줄에 날것 `::`가 있으면 그 줄이 새 카드로 잘립니다.** 답변 본문에서 `::`를 쓸 일이 있으면 반드시 백틱이나 ```펜스```로 감싸세요. (개발자 노트에선 `::`가 거의 항상 코드이므로 자연스럽게 안전합니다.)
- 한 줄 카드(`질문?::답변`)는 기존과 동일하게 동작합니다.

## 3. 가져오기

앱에서 **설정 탭 → 데이터 소스 → `Obsidian Vault 가져오기`** 를 누릅니다.

### 방법 A — Vault 폴더 선택 (권장, 데스크톱·안드로이드 크롬)

`Vault 폴더 선택`을 누르고 Obsidian Vault 폴더를 지정하면, 하위 폴더까지 모든 `.md`를 훑어 카드를 찾습니다.

### 방법 B — .md 파일 직접 선택 (모든 환경, iPhone 포함)

`.md 파일 선택`을 누르고 카드가 든 노트 파일을 고릅니다. 여러 개를 한 번에 선택할 수 있습니다.

> iPhone Safari는 폴더 선택을 지원하지 않으므로 방법 B를 사용하세요.

선택하면 **몇 개 덱 / 몇 장 카드**가 인식됐는지 미리보기가 뜹니다. 숫자를 확인하고 `N장 가져오기`를 누르면 끝입니다.

---

## 4. 노트를 고친 뒤 (갱신)

Obsidian에서 카드를 추가·수정했다면 **다시 가져오기**만 하면 됩니다.

- **새 카드** → 추가됨
- **삭제한 카드** → 함께 삭제됨
- **내용이 바뀐 카드** → 질문/답변만 갱신, **복습 일정(SM-2)은 그대로 유지**
- **그대로인 카드** → 변화 없음

즉, 다시 가져와도 그동안 쌓인 복습 진도는 사라지지 않습니다.

---

## 5. 데모로 되돌리기

내 Vault를 가져온 뒤 다시 샘플 카드로 돌아가려면 **설정 → 데이터 소스 → `데모 카드로 되돌리기`** 를 누릅니다. (가져온 카드와 그 일정은 비워지고 번들 샘플이 복원됩니다.)

## 문제 해결

| 증상 | 확인할 것 |
|------|-----------|
| "카드를 찾지 못했습니다" | 노트에 `## Self-Test Anchors` 줄이 있는지, 그 아래 `#flashcard/...` 태그와 `질문?::답변` 줄이 있는지 |
| 일부 카드만 들어옴 | 카드 줄에 `::`가 빠졌거나, 해당 카드가 `### 티어` 헤딩 아래에 없을 수 있음 |
| 티어가 안 맞음 | `### Foundation/Mechanism/Diagnosis` 철자 확인 (그 외 헤딩은 티어로 인식 안 됨) |
| 폴더 선택 버튼이 없음 | 브라우저가 폴더 선택 미지원(예: iOS) — `.md 파일 선택` 사용 |
| 다른 기기와 동기화 안 됨 | 데이터는 기기·브라우저별 로컬 저장. 기기마다 각각 가져오면 됨 |
